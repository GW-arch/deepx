## 핵심 문제

**CPU 버전(MediaPipe, TFLite float32)은 손과 손가락을 정확히 검출하지만, NPU 버전(INT8 양자화)은 제대로 검출하지 못한다.**

구체적으로 두 가지 정확도 문제가 있다:

1. **Palm detection**: INT8 양자화 시 score head가 완전히 파괴되어 NPU에서 손 검출 자체가 불가. CPU TFLite(float32)로 우회 중.
2. **Hand landmark**: NPU(.dxnn INT8)로 실행하면 **특정 손(Left hand)의 손가락 랜드마크가 크게 틀어짐**. dataset 프레임 기준 측정 결과:

   | 손 | 전체 21 랜드마크 평균 오차 | 손끝 5개 평균 오차 | 최대 오차 | 비고 |
   |----|------------------------:|------------------:|---------:|------|
   | Right hand | 0.015 | 0.017 | 0.030 | 정상 (wrist dx=+0.024) |
   | Left hand | **0.077** | **0.087** | **0.160** | 엄지 0.143, 검지 0.132 — **사용 불가 수준** |

   Left hand의 엄지·검지 tip이 CPU 대비 x방향으로 +0.12~0.13 (화면의 12~13%) 벗어남. 이는 INT8 양자화 오차가 특정 손 방향/ROI 크롭에서 증폭되는 것으로 추정.

결과적으로 NPU를 활용하려 해도 (1) palm은 CPU 고정이라 병목(~95ms), (2) hand landmark도 한쪽 손의 정확도가 실용 수준에 미치지 못하는 상태.

## 3. 해결 / 미해결 정리 및 다음 할 일

### 해결된 문제

- **P3 (ROI 드리프트)**: `_PALM_REDETECT_EVERY = 0`으로 해결. 정확도는 완벽하나 속도 희생.
- **시각화**: 풀스크린, 악기 다이어그램, 타격 피드백 등 시각화 전반 완성.
- **피아노 기본 모드**: 정적 JSON 매핑으로 변경 완료.
- **cpu-baseline 기준선**: 구현 완료, NPU 효과 비교 가능.

### 미해결 문제

- **P1 (NPU hand landmark 정확도)**: Left hand 손가락 오차가 0.13+로 실용 불가 수준. Right hand는 정상. INT8 양자화 오차가 특정 ROI/손 방향에서 증폭되는 원인 조사 필요. 가능한 방향:
  - ROI crop/역변환 코드에 좌우 비대칭 버그가 있는지 확인
  - 동일 패치를 TFLite(float32) vs .dxnn(INT8)에 넣어 모델 출력 자체를 비교
  - 보정 데이터셋(calibration images) 품질 개선으로 양자화 정확도 향상 시도
- **P4 (npu-full 레이턴시)**: Palm이 병목. 해결 방향:
  - **Palm 모델 경량화**: 더 작은 palm detection 모델 탐색 (MobileNet-V2 SSD 등)
  - **Palm NPU 부분 양자화**: score head만 CPU에서 float32로 실행하는 하이브리드 양자화 (DX-COM이 지원하는지 확인 필요)
  - **Palm skip 복구**: 드리프트를 보정하는 방식으로 (예: 매 N프레임 palm 실행 + 드리프트 감지 시 즉시 재실행)
  - **비동기 palm**: palm detection을 별도 스레드에서 실행하여 hand landmark와 병렬화
- **P5 (NPU가 CPU보다 느림)**: hand landmark 단독 비교에서 NPU ~8ms vs CPU ~5ms. DX-RT 초기화/디스패치 오버헤드 또는 INT8 conv가 이 모델 크기에서 이점이 없는 것일 수 있음. 프로파일링 필요.

### 다음 할 일 (우선순위)

1. **NPU hand landmark 정확도 개선** — Left hand 손가락 오차(0.13+)의 원인을 파악하고 수정. ROI crop/역변환 비대칭 버그 점검, 동일 패치에 대한 TFLite vs .dxnn 출력 비교, DX-COM 보정 데이터셋 품질 개선 등.
2. **사전 생성 데이터셋 활용** — 라이브 카메라 없이도 정확도·성능을 반복 측정할 수 있도록, 다양한 손 자세·조명의 프레임을 미리 촬영하여 `dataset/`에 저장. CPU vs NPU 비교, 회귀 테스트, 벤치마크에 일관된 입력으로 사용.
3. **비동기 palm detection 실험** — palm을 별도 스레드에서 실행하여 hand landmark와 파이프라이닝. palm ~95ms 동안 이전 palm 결과의 ROI로 hand landmark를 계속 실행하면 실질 프레임 레이턴시를 ~8ms로 줄일 수 있음.
4. **Palm skip + 드리프트 보정** — EMA나 칼만 필터로 ROI 드리프트를 감지·보정하면서 palm skip을 부분 복구.
5. **NPU hand landmark 프로파일링** — `dx_engine` 타이밍 API로 실제 NPU 연산 시간 vs 호스트 오버헤드 분리.
6. **최종 보고서용 벤치마크** — 4개 백엔드(cpu, cpu-baseline, npu-full, npu)의 정식 벤치마크 (N≥100 프레임, 평균/P95/stddev). 사전 생성 데이터셋 기반으로 재현 가능한 측정.
