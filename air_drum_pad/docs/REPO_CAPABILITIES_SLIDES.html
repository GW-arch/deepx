Repository Capability Review

deepx / air_drum_pad 실험 가능성 분석

이 레포는 air drum과 piano 데모를 실행하고, CPU/TFLite/NPU 백엔드를 비교하며, 데이터셋 기반 벤치마크와 NPU landmark 보정까지 수행할 수 있는 상태다. 단, NPU 실험은 DX-RT service와 모델 자산 상태가 실험의 관문이다.

Executive Summary

실험 가능한 상태인가?

결론은 가능이다. CPU 기반 데모, 피아노/드럼 매핑, 단위 테스트, palm decode, CPU-baseline dataset benchmark는 바로 실행 가능하다. NPU-full 실험은 모델 파일은 로컬에 있으나 현재 확인 시 DX-RT service가 실행 중이 아니어서 service 기동 또는 NPU 런타임 복구가 필요했다.

Interaction Model

고정 패드가 아니라 손가락 움직임을 연주로 해석

타격 조건

  • 손끝 y 방향 하강 속도 vy가 임계 이상
  • MCP/PIP/TIP 계열 관절 각속도가 임계 이상
  • 같은 손가락 cooldown과 최소 이동량으로 jitter 억제

사운드 매핑

  • 양손 x 다섯 손가락 = 10 slot
  • 드럼은 16개 합성 percussion 중 slot 매핑
  • 피아노는 음명 10개를 합성 tone으로 미리 렌더링

Pipeline

`npu-full` 내부 처리 흐름

01

RGB frame

OpenCV 카메라 프레임을 RGB로 변환한다.

02

Palm TFLite

192x192 letterbox 입력으로 palm box와 keypoint를 얻는다.

03

ROI warp

Palm keypoint로 회전 사각형을 만들고 224x224 hand patch를 생성한다.

04

Hand DXNN

NPU에서 21 landmark를 추론하고 원본 좌표로 역변환한다.

05

Strike

손끝 속도와 관절 각속도로 사운드 trigger를 만든다.

이 구조 덕분에 CPU-baseline과 npu-full은 같은 palm/ROI 경로를 공유한다. 따라서 hand landmark 단계만 CPU TFLite와 NPU `.dxnn`으로 비교할 수 있다.

Palm Bottleneck

Palm `.dxnn`은 속도는 빠르지만 실사용 불가 판정

좋은 점

Palm NPU 추론은 약 12 ms로 CPU TFLite 약 95 ms보다 빠르다.

문제

INT8 양자화 후 score head가 파괴되어 손 검출 score가 신뢰 불가능해진다.

현재 결론

Palm은 TFLite CPU float32 고정, Hand landmark만 NPU로 실행한다.

aggressive partitioningscore head 문제 해결 안 됨

Environment note
Risks

실험 결과를 해석할 때의 리스크

NPU runtime availability

모델 파일만으로 충분하지 않다. `dxrt-cli`, service, NPU reset, `dx_engine` ABI가 맞아야 한다.

Calibration overfit

현재 correction은 training dataset에서 좋아진 값이다. hold-out이 없으면 demo 품질을 보장할 수 없다.

Palm bottleneck

Palm CPU TFLite가 전체 지연의 큰 비중이다. palm skip/async는 정확도와 trade-off가 있다.

E2E latency gap

Offline benchmark는 vision loop 기준이다. 카메라 입력부터 오디오 출력까지의 E2E 측정은 별도다.

Ignored assets

dataset과 vendor model은 Git ignore 대상이다. 실험 전달 시 자산 패키징 전략이 필요하다.

Unwired utilities

거리 기반 dynamic piano helper는 존재하지만 현재 `main.py`의 live path에는 연결되어 있지 않다.

Final Judgement

실험 레포로는 충분히 준비되어 있다

이 레포는 “데모 앱”과 “실험 장치”를 모두 갖고 있다. 바로 보여줄 수 있는 것은 CPU 기반 air drum/piano와 dataset replay이고, 핵심 연구 가치는 CPU-baseline 대 NPU-full 비교, palm 병목 최적화, NPU landmark 보정 검증에 있다.

강점

반복 가능한 benchmark, 문서화된 모델 파이프라인, 테스트 루프가 있다.

주의

NPU runtime과 ignored asset 상태가 실험 재현성의 핵심 관문이다.

다음

DX-RT service 복구 후 hold-out dataset으로 correction/async/palm skip을 검증한다.
